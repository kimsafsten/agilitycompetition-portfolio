# Publish Checklist (Public Portfolio Repo)

## Keep public
- README, feature descriptions, architecture overview
- Screenshots and demo links
- High-level technical decisions and learnings

## Keep private
- Full production source code
- Sensitive business logic details
- Secrets, keys, tokens, connection strings
- Internal endpoints and private admin credentials

## Before publishing
1. Verify no `.env` or config files with secrets are included.
2. Verify screenshots do not expose personal data or admin credentials.
3. Add a restrictive license (or no license) if you do not want reuse.
4. Confirm demo account has limited permissions.
5. Test all public links.
